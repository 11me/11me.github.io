How Do Containers Work?

Introduction

To understand how docker makes its magic, first one have to understand some of the features of the Linux kernel. These features called namespaces and cgroups. These features allow each process to have its own system view and run in isolation from other processes.

You can get a list of available namespaces by running the following command.

$ ls /proc/$$/ns
cgroup ipc mnt net pid pid_for_children time time_for_children user uts

You can imagine namespaces as a boxes for processes, so each process contained within a set of boxes and if the process goes crazy and deletes the whole filesystem it will not affect other processes. Namespaces and cgroups deserve their own topic and I am not going to go deep into details.

Now we know, that Linux kernel can isolate processes and this what we call containers. This is actually it. AND WHO NEEDS DOCKER NOW HU?!

Let’s create a container.

  1. Create a root directory for a process that will be jailed1.
$ mkdir -p croot/bin
  1. Copy bash into croot/bin directory.
$ cp $(which bash) chroot/bin

  1. croot stands for container root. ↩︎
